# Контекст Бэкенда: Siege Architect
Суть: Центральный узел (Hub) кроссплатформенной SaaS-системы для управления PvP-осадами.
Стек: Laravel 11+, PostgreSQL, PHP 8.3+, Sanctum.

# Архитектурные правила:
- Service/Action Pattern: Вся бизнес-логика строго в App\Actions. Контроллеры только принимают и возвращают данные.
- Single Source of Truth: Верификация прав на уровне API (Policies). Ни один клиент не может обойти эти права.
- Backend-First: Разработка начинается с миграции и Action-класса.
- API: Все эндпоинты в префиксе /api/v1/. Валидация через FormRequest.
- Event-Driven Sync: При изменениях бэкенд инициирует push-уведомления (через очереди) в подключенные интеграции.

# Принципы написания кода:
- Приоритет на скорость разработки и читаемость кода. Избегай излишних абстракций.
- Не используй Enum, применяй простые константы или строки.
- Не пиши Unit-тесты для экономии времени, если это не запрошено отдельно.
- Если написание тестов необходимо, используй строго PHPUnit (Pest запрещен).

Обязательное требование: при создании или изменении API эндпоинтов, всегда обновляй соответствующий файл контракта в директории ../_docs/. Контракт должен содержать URL, метод, примеры JSON и TypeScript-интерфейсы.

После написания кода пишешь краткое резюме проделланой работы в `../tasks/botPhp/released/название_задачи.md`
Ты работаешь в консоли windows и не можешь использовать ls например
